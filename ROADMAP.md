## v0.3.0 — 生产就绪（目标：2026-07）

### 目标：从"能跑"到"能部署"

#### 1. 持久化全面接入
- [ ] `eneros-api` 启动时从配置文件读取 `storage_backend` 和 `sqlite_path`，自动创建 `TimeSeriesEngine::with_sqlite()`
- [ ] `eneros-gateway` 命令历史持久化到 SQLite（重启后可审计）
- [ ] `eneros-agent` Agent 记忆系统接入 SQLite 后端
- [ ] 添加数据库迁移脚本和 schema 版本管理

#### 2. 配置体系
- [ ] 统一配置文件（TOML/YAML），覆盖 API 端口、数据库路径、设备连接参数、约束阈值
- [ ] 环境变量覆盖支持（`ENEROS_*`）
- [ ] 配置校验和启动时友好报错

#### 3. 可观测性
- [ ] 结构化日志（JSON 格式），支持日志级别动态调整
- [ ] Prometheus metrics 端点（命令执行延迟、队列深度、约束违规数、Agent 决策延迟）
- [ ] 分布式追踪（trace_id 贯穿 API→Pipeline→Gateway→Device）

#### 4. 安全加固
- [ ] API 认证（JWT 或 mTLS）
- [ ] 命令执行权限审计日志
- [ ] 设备连接 TLS 加密

## 版本发布节奏

| 节奏 | 说明 |
|------|------|
| **次版本** (0.X.0) | 每 4-6 周一个次版本，包含新功能 |
| **修订版本** (0.0.X) | 按需发布，仅包含 Bug 修复 |
| **主版本** (1.0.0) | 达到生产级稳定性后发布 |

## 优先级原则

1. **P0 — 安全/数据完整性**：数据丢失、安全漏洞优先修复
2. **P1 — 核心功能**：影响主流程的 Bug 和功能缺失
3. **P2 — 体验优化**：性能、可观测性、文档
4. **P3 — 新功能扩展**：新协议、新分析模块、新 Agent 策略
